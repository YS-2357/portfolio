# Compare-AI 프로젝트 요약

- 문제: 같은 질문이라도 LLM마다 답변 품질과 톤이 달라 선택 비용이 큼
- 접근: 멀티 모델 팬아웃 호출 + 스트리밍 수집 + 공통 포맷 정규화
- 역할: 백엔드 파이프라인 설계, 응답 비교 로직, UI 연결, 배포
- 결과: 모델 비교 시간을 단축하고, 응답 선택 기준을 일관되게 제공

## 핵심 구현

- 다중 모델 요청을 비동기로 병렬 처리
- 모델별 응답 지연/실패를 고려한 안정적 수집 로직
- 비교 가능한 기준(구조/핵심성/길이 등)으로 요약
- 프론트에서 빠르게 확인 가능한 카드형 결과 구성
